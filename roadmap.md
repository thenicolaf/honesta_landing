## 🔐 Admin Panel — `/admin`

### Защита доступа

> Рекомендуемый подход: отдельная роль `admin` в Supabase Auth + middleware в Next.js.

```
1. В Supabase: таблица users с полем role ('admin' | 'customer')
2. В Next.js middleware: проверка роли при каждом запросе к /admin/*
3. Если роль не admin — редирект на 404 или главную страницу
4. Отдельная страница входа: /admin/login (не связана с клиентским логином)
5. Опционально: ограничение по IP-адресу на уровне Supabase RLS
```

Обычный пользователь, попав на `/admin`, увидит 404 — он даже не узнает, что такой роут существует.

## 🤖 ManyChat — Автоматизация мессенджеров

### Что это и зачем

ManyChat — платформа для чат-ботов в Instagram Direct и WhatsApp. Позволяет клиенту взаимодействовать с магазином прямо в мессенджере: смотреть каталог, узнавать статус заказа, получать уведомления — без захода на сайт.

В Дубае WhatsApp и Instagram — основные каналы коммуникации, поэтому автоматизация этих каналов напрямую влияет на конверсию.

### Цепочка автоматизации

```
Клиент (Instagram / WhatsApp)
        ↓
ManyChat — приём сообщений, flow-логика, отправка ответов
        ↓
Next.js API Routes — бизнес-логика, валидация, маршрутизация
        ↓
Supabase — товары, заказы, клиенты, лиды
        ↑
Supabase Webhooks — триггеры при смене статуса заказа
        ↑
Next.js → ManyChat API — уведомления клиенту по инициативе сервера
```

> **n8n не нужен** — Next.js полностью покрывает роль оркестратора в этой цепочке.

### За что отвечает каждое звено

| Звено | Ответственность |
|-------|----------------|
| **ManyChat** | Приём сообщений, flow-сценарии, кнопки, карусели, рассылки по подписчикам |
| **Next.js API** | Бизнес-логика, защита по токену, запросы к Supabase, вызов ManyChat API |
| **Supabase** | Хранение данных, Database Webhooks при смене статуса заказа |
| **ManyChat API** | Исходящие уведомления клиенту по инициативе сервера |

### API эндпоинты для реализации

- [ ] `POST /api/manychat/search-products` — поиск товаров по запросу / категории
- [ ] `GET /api/manychat/order-status` — статус заказа по номеру
- [ ] `POST /api/manychat/save-lead` — сохранить контакт из бота в Supabase
- [ ] `POST /api/manychat/notify-subscriber` — отправить уведомление клиенту
- [ ] `POST /api/manychat/broadcast` — рассылка по сегменту аудитории

### Flow-сценарии

- [ ] **Каталог** — клиент пишет "catalog" → бот показывает категории → карусель товаров с ценами
- [ ] **Статус заказа** — клиент вводит номер → бот возвращает актуальный статус из Supabase
- [ ] **Уведомления о заказе** — смена статуса в Admin Panel → Supabase Webhook → сообщение в WhatsApp
- [ ] **Брошенная корзина** — cron job каждые 2 часа → напоминание + промокод на 5%
- [ ] **Сезонные рассылки** — Рамадан, Ид аль-Адха — рассылка по базе с актуальными товарами

### Новые таблицы в Supabase

```sql
-- Контакты из Instagram / WhatsApp
create table leads (
  id uuid primary key default gen_random_uuid(),
  manychat_id text unique,
  name text,
  phone text,
  instagram_handle text,
  language text default 'en',       -- en | ru | ar
  tags text[],                       -- ['vip', 'wholesale', 'ramadan']
  created_at timestamptz default now()
);

-- Лог уведомлений — чтобы не спамить
create table notification_log (
  id uuid primary key default gen_random_uuid(),
  lead_id uuid references leads(id),
  message_type text,                 -- 'order_status' | 'promo' | 'abandoned_cart'
  sent_at timestamptz default now()
);
```

Database Webhook на `orders.status UPDATE` → `POST /api/manychat/notify-subscriber`.

## 🧠 AI Агент — Интеллектуальный слой обработки запросов

### Что это и зачем

ManyChat из коробки умеет только реагировать на ключевые слова и кнопки. AI агент — это следующий уровень: клиент пишет свободным текстом на любом языке, а бот понимает намерение, знает каталог и историю клиента, и отвечает как живой консультант.

ManyChat остаётся транспортом — принять и отдать сообщение. Весь интеллект живёт в Next.js + Claude API.

### Архитектура

```
ManyChat → POST /api/manychat/ai-agent
                    ├── Supabase: история диалога + профиль клиента
                    ├── Supabase: каталог товаров (контекст для агента)
                    ├── Claude API: генерация умного ответа
                    └── Supabase: сохранить сообщение в ai_conversations
```

### Что умеет AI агент

- [ ] Понимать свободный текст на EN / RU / AR
- [ ] Рекомендовать товары по описанию ("хочу что-то сладкое, без сахара, граммов 500")
- [ ] Подбирать подарочные наборы ("на подарок маме, она любит финики")
- [ ] Отвечать на вопросы о составе, хранении, Halal-статусе, стране происхождения
- [ ] Персонализировать ответы на основе истории заказов клиента
- [ ] Подсказывать до бесплатной доставки ("добавьте ещё 50г — и доставка бесплатная")
- [ ] Помнить контекст диалога в рамках сессии

### Новый эндпоинт

- [ ] `POST /api/manychat/ai-agent` — принять сообщение → собрать контекст → Claude API → вернуть ответ

### Новая таблица в Supabase

```sql
-- Память агента — история диалога по каждому клиенту
create table ai_conversations (
  id uuid primary key default gen_random_uuid(),
  lead_id uuid references leads(id),
  messages jsonb,    -- [{ role: 'user' | 'assistant', content: string }]
  updated_at timestamptz default now()
);
```

### Выбор модели

| Модель | Рекомендация |
|--------|-------------|
| **Claude Sonnet** | Оптимально — лучшее понимание контекста, естественный тон, хорошо работает с арабским |
| GPT-4o mini | Альтернатива — дешевле, чуть слабее в нюансах |
